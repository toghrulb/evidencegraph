# API

Phase 1 exposes health, collection, and original-document storage endpoints. Search, questions, comparison, and evaluation remain unavailable.

## Health

- `GET /health/live` returns `200 {"status":"ok"}` while the process can serve requests.
- `GET /health/ready` returns `200 {"status":"ready"}` during the FastAPI lifespan and `503 {"status":"not_ready"}` before startup or during shutdown.

Readiness is still a process-lifecycle signal. Compose separately gates startup on PostgreSQL, Redis, and MinIO health checks.

## Collections

### `POST /api/v1/collections`

Creates a collection and returns `201`. The JSON body contains a required `name` and optional `description`.

```json
{
  "name": "Retrieval papers",
  "description": "Papers to compare"
}
```

### `GET /api/v1/collections`

Returns all collections newest first.

### `GET /api/v1/collections/{collection_id}`

Returns one collection or `404`.

### `DELETE /api/v1/collections/{collection_id}`

Deletes the collection, its document rows, and every associated original PDF. Returns `204`, `404` for a missing collection, or `503` if object storage cannot complete the deletion.

## Documents

### `POST /api/v1/collections/{collection_id}/documents`

Accepts `multipart/form-data` and returns `201` document metadata.

- `file`: required PDF
- `title`: optional string; defaults to the safe filename stem
- `authors`: optional repeated string field
- `publication_year`: optional integer from 1000 through next year

Validation is streamed and enforces the configured byte limit, `application/pdf` MIME type, a `.pdf` display filename, and a `%PDF-` signature. The source filename is reduced to a sanitized basename; the object key is generated by the server. A SHA-256 checksum prevents the same bytes from being added twice to one collection (`409`). The same PDF may exist in a different collection.

The response does not expose the internal storage key. It includes:

```json
{
  "id": "<uuid>",
  "collection_id": "<uuid>",
  "title": "A useful paper",
  "authors": ["Ada Lovelace"],
  "publication_year": 2025,
  "source_filename": "paper.pdf",
  "checksum": "<sha256>",
  "status": "uploaded",
  "page_count": null,
  "error_message": null,
  "created_at": "<timestamp>",
  "updated_at": "<timestamp>"
}
```

After the metadata transaction commits, a Celery job is queued. The worker atomically and idempotently changes `uploaded` to `processing`. Phase 1 performs no parsing, so it never changes the document to `ready`. A queue-dispatch failure is persisted as `failed` with a controlled error message.

### `GET /api/v1/collections/{collection_id}/documents`

Returns the collection's document metadata newest first, or `404` when the collection does not exist.

### `GET /api/v1/documents/{document_id}`

Returns one document's metadata or `404`.

### `GET /api/v1/documents/{document_id}/status`

Returns the polling subset `id`, `status`, `error_message`, and `updated_at`.

### `GET /api/v1/documents/{document_id}/file`

Streams the original PDF inline from MinIO. Returns `404` for missing metadata or `503` when the stored object is unavailable.

### `DELETE /api/v1/documents/{document_id}`

Deletes the original PDF and metadata, returning `204`. Deleting a missing document returns `404`.

Interactive OpenAPI documentation is available at `/docs` while the backend is running.
